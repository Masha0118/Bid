# 경매 종료 시연 대본

## 촬영 순서

1. 종료 직전 상품 상세와 마지막 입찰을 보여줍니다.
2. 종료 시각 이후 낙찰자·낙찰가와 상품 상태가 바뀌는지 확인합니다.
3. 낙찰자의 거래 목록에서 새 거래를 확인합니다.
4. 판매자와 구매자 알림을 각각 확인합니다.
5. 거래 채팅방이 생성되거나 기존 정책에 따라 연결되는지 확인합니다.
6. 같은 종료 요청을 개발 환경에서 재시도할 수 있다면 거래가 하나인지 확인합니다.

## 실패 시나리오 별도 촬영

Main API 후속 처리를 의도적으로 실패시키는 안전한 개발 환경이 있을 때만 촬영합니다. 실패 뒤 Redis 상태가 남고 재시도로 완료되는지 확인하며, 운영 데이터나 실제 내부 주소는 노출하지 않습니다.
