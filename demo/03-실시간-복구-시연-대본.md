# WebSocket 복구 시연 대본

## 촬영 순서

1. 두 계정 중 한 화면에서 입찰해 다른 화면이 새로고침 없이 바뀌는지 확인합니다.
2. 개발용 네트워크 도구로 한 화면의 WebSocket 연결을 끊습니다.
3. 연결이 끊긴 동안 다른 계정에서 입찰합니다.
4. 네트워크를 복구하고 재연결 상태를 확인합니다.
5. 최신 상태가 REST 재조회 또는 이어진 버전 처리로 복구되는지 확인합니다.
6. 입찰 목록에 같은 항목이 중복되지 않는지 확인합니다.

## 반드시 확인할 점

- 현재 코드는 버전 누락 시 전체 재조회를 요청합니다.
- 실제 WebSocket 연결을 끊고 중간 이벤트를 빠뜨리는 재현 절차는 촬영 환경에서 먼저 검증해야 합니다.
- 재현이 불안정하면 누락 복구를 성공 장면처럼 편집하지 말고 단위 테스트 영상으로 대체합니다.
