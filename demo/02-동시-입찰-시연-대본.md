# 동시 입찰 시연 대본

## 준비

- 같은 상품에 접속한 서로 다른 테스트 계정 2개
- 양쪽 화면에 같은 현재가가 보이는 상태
- 필요하면 개인정보를 제외한 서버 지표 화면

## 1분 촬영 순서

1. 두 화면의 상품과 현재가가 같음을 보여줍니다.
2. 서로 다른 유효 금액을 입력합니다.
3. 카운트다운 뒤 거의 동시에 입찰 버튼을 누릅니다.
4. 두 화면에서 최종 최고가와 최고 입찰자가 일치하는지 확인합니다.
5. 입찰 목록이 중복되지 않고 버전 순서가 이어지는지 보여줍니다.
6. 서버 로그 대신 가능하면 비식별 처리된 처리 지표와 감사 이벤트 건수를 보여줍니다.

## 판정 기준

- 한 요청만 이전 상태를 기준으로 성공해야 함
- 두 화면의 최종 가격·입찰자·종료 시각이 같아야 함
- 실패 응답이 있어도 최종 상태가 성공한 입찰과 일치해야 함

실제 부하 성능 수치는 재현 가능한 테스트 보고서가 준비되기 전에는 영상 자막에 넣지 않습니다.
