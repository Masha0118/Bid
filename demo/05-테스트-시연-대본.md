# 자동화 테스트 시연 대본

이번 문서 작성 과정에서는 원본 프로젝트 읽기 전용 조건 때문에 테스트를 실행하지 않았습니다. 별도 복제 환경에서 다음 순서로 촬영합니다.

1. 경매 서버의 입찰 규칙 테스트: 판매자 입찰, 최소가, 시간 조건
2. 자동 연장과 Redis 상태 변경 테스트
3. 중복 종료와 Main API 실패 시 상태 유지 테스트
4. Flutter의 중복·누락·늦은 REST 응답 테스트
5. Kafka ACK·중복 소비·실패 보관 테스트
6. Batch 위험 계산 테스트

## 촬영 체크리스트

- 실행 명령과 프로젝트명만 보이고 사용자 경로는 가렸는가
- 환경변수와 비밀값이 출력되지 않는가
- 전체 성공 건수는 실제 완료 결과와 일치하는가
- 건너뛴 통합 테스트가 있다면 이유를 자막에 표시했는가
- 실패를 숨기기 위해 소스나 설정을 즉석에서 수정하지 않았는가
