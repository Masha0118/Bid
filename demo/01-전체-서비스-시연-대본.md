# 전체 서비스 시연 대본

코드에서 확인한 사용자 흐름을 기준으로 한 3~5분 촬영안입니다. 실제 재현되는 장면만 최종 영상에 사용합니다.

## 준비

- 판매자·입찰자 테스트 계정 2개와 관리자 테스트 계정
- 시작이 임박한 테스트 경매 상품
- 화면 녹화에서 주소·연락처·토큰·내부 주소 숨김

## 촬영 순서

1. 로그인 후 홈과 상품 상세를 짧게 보여줍니다.
2. 판매자 계정에서 상품과 경매 조건을 등록합니다.
3. 경매 시작 뒤 두 계정의 상세 화면을 나란히 배치합니다.
4. 입찰하고 새로고침 없이 가격과 최고 입찰 정보가 함께 바뀌는지 보여줍니다.
5. 마감 임박 입찰로 종료 시각이 연장되는지 확인합니다.
6. 종료 뒤 낙찰 결과와 상품 상태를 확인합니다.
7. 낙찰 거래, 채팅방과 알림으로 이동합니다.
8. 관리자 화면에서 해당 상품·신고·거래 조회 위치를 짧게 보여줍니다.

## 설명할 기술 포인트

- 입찰 검증과 상태 변경은 Redis Lua에서 한 번에 처리
- WebSocket은 변경 정보와 버전을 전달
- 낙찰 결과가 성공한 뒤 거래와 런타임 상태 정리가 이어짐

광고 심사와 회원 제재는 별도 데이터 준비가 필요하므로 전체 흐름 영상에 무리하게 포함하지 않고 스크린샷 또는 별도 영상으로 분리합니다.
