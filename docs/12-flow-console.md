# Bid On Flow Console

## 제작 목적

실제 Flutter 화면과 운영 소스코드를 공개하지 않고 실시간 경매 백엔드의 데이터 변경과 처리 순서를 직접 확인할 수 있도록 만든 공개용 시각화 도구입니다. 비공개 원본 프로젝트와 공개 문서에서 확인한 흐름을 브라우저 안에서 정적으로 재생합니다.

## 실제 프로젝트와의 관계

| 구분 | Flow Console 기준 |
|---|---|
| 처리 단계 | 비공개 원본 프로젝트와 공개 문서를 기준으로 재구성 |
| 사용자 정보 | User A, User B 등 공개용 가상 사용자 |
| 금액·시간·Version | 처리 전후를 설명하기 위한 가상 값 |
| UI | 실제 앱과 관계없는 기술 시연 화면 |
| 네트워크 | 실제 운영 서버와 연결되지 않는 정적 재생 |
| 로그 | 실제 로그 원문이 아닌 설명용 이벤트 |
| 성능 수치 | 포함하지 않음 |

## 포함된 시나리오

| 시나리오 | 보여주는 내용 | 구현 근거 | 검증 상태 |
|---|---|---|---|
| 정상 입찰 | 검증·Redis 상태 변경·후속 전달 | 코드에서 구현 확인 | 테스트 코드 확인 · 흐름 재구성 |
| 동시 입찰 | 요청 판정과 최종 상태 결정 | 코드와 테스트 코드 확인 | 테스트 코드 확인 · 흐름 재구성 |
| 자동 연장 | 종료 시각과 Version 변경 | 코드와 테스트 코드 확인 | 테스트 코드 확인 · 흐름 재구성 |
| 이벤트 누락 | 전체 상태 재조회 판단 | 코드와 테스트 코드 확인 | 테스트 코드 확인 · 흐름 재구성 |
| 중복 종료 | 잠금과 중복 거래 방지 | 코드와 테스트 코드 확인 | 테스트 코드 확인 · 흐름 재구성 |
| 거래 재처리 | 실패 시 상태 유지와 후속 처리 | 코드와 테스트 코드 확인 | 흐름 재구성 · 추가 검증 필요 |

Flow Console 자체의 Query Parameter 동작은 자동화 테스트를 실행했고 정적 빌드도 확인했습니다. 위 표의 원본 프로젝트 테스트는 이번 작업에서 실행하지 않았습니다.

## 화면 구성

- 시나리오 선택과 Query Parameter 딥링크
- 시스템 처리 흐름과 원자 처리·후속 처리 구분
- 처리 전후 상태
- 이벤트 순서
- 구현 및 검증 근거
- 관련 공개 문서

## 공개하지 않는 정보

- 실제 API 주소, Redis Key, Kafka Topic, RabbitMQ Queue
- 인증 정보와 사용자 데이터
- 운영 로그와 실제 Flutter UI
- 비공개 소스코드와 성능 수치

## 실행 방법

```bash
cd flow-console
npm ci
npm run dev
```

딥링크는 `?scenario=concurrent-bid` 형식입니다. 유효한 시나리오 ID와 Pages 하위 경로 대응 방식은 [Flow Console README](../flow-console/README.md#query-parameter-딥링크)에 정리했습니다.

## 테스트 및 빌드

```bash
npm run type-check
npm run test
npm run build
```

이번 공개 저장소에서 위 명령을 실행해 TypeScript 검사, 딥링크 단위 테스트와 Vite 정적 빌드를 확인합니다. 실행 결과가 달라지면 이 문서의 상태도 함께 갱신해야 합니다.

## 현재 한계

- 실제 시스템 모니터링 화면이나 실시간 서버 연결이 아닙니다.
- 처리 시간은 성능 측정값이 아니라 시각화를 위한 상대 시간입니다.
- 원본 프로젝트 테스트 코드는 확인했지만 이번 공개 저장소에서 실행하지 않았습니다.
- 실제 장애 환경, 네트워크, 동시 부하와 다중 인스턴스 통합 검증을 대신하지 않습니다.
