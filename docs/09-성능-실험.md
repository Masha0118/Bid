# 성능 실험

[웹에서 고급 성능 보기](https://bid-on-performance-lab.pages.dev/) · [테스트 실행 환경](../diagrams/test-environment.mmd)

## 기본 체험 설정

기본 체험에서는 2~10명의 가상 참가자와 다음 입찰 속도를 선택합니다.

- 느림: 2초 간격
- 보통: 1초 간격
- 빠름: 0.4초 간격

순차, 경쟁, 유효 범위 분산 패턴으로 요청 금액을 구성합니다. 이 설정은 한 경매의 짧은 동시 입찰 상황을 이해하기 위한 범위입니다.

## 고급 성능 설정

`고급 성능 보기`는 k6를 이용해 가상 사용자 수, 실행 시간, 증가 시간, 요청 간격과 WebSocket 구독 수를 제한 범위 안에서 설정합니다. 브라우저가 임의 스크립트나 대상 URL을 전달하지 않고 서버가 허용한 시나리오만 실행합니다.

## 결과 지표

| 지표 | 의미 |
|---|---|
| 시도한 입찰 | 서버로 보낸 전체 입찰 요청 |
| 반영된 입찰 | 경매 상태 변경에 반영된 요청 |
| 조건상 미반영 | 현재가·최고 입찰자·경매 상태 같은 정상 조건으로 반영되지 않은 요청 |
| 시스템 오류 | 정상 경매 조건 밖에서 처리하지 못한 요청 |
| 응답 시간 초과 | 제한 시간 안에 응답하지 않은 요청 |
| 평균 응답 시간 | 완료 요청의 평균 처리 시간 |
| p95 | 완료 요청의 95%가 이 값 이하에서 처리된 응답 시간 |
| p99 | 고급 결과에서 제공될 때 확인하는 상위 지연 시간 |
| 초당 처리량 | 실행 구간에서 초당 완료된 요청 수 |

조건상 미반영은 시스템 실패와 분리합니다. 경쟁 패턴에서는 여러 참가자가 같은 최소가를 기준으로 요청하므로 정상 규칙에 따른 미반영이 발생할 수 있습니다.

## 종료와 정합성 확인

실행이 끝나면 짧은 재확인 구간을 두고 최종 상태를 비교합니다.

- 가장 높은 반영 금액과 최종 현재가
- 해당 참가자와 최종 최고 입찰자
- 반영된 입찰 수와 입찰 횟수
- 참가자 카드와 서버의 Version

확인 정보가 수집되지 않았거나 제한 시간 안에 수렴하지 않으면 성공으로 간주하지 않고 별도 상태로 표시합니다.

## 결과 해석 시 주의점

표시되는 결과는 해당 테스트 환경의 전체 실행 결과입니다. 서버만의 절대적인 최대 처리 한계를 의미하지 않습니다. 부하 발생기, 경매 실행 제어와 서버가 같은 장비 또는 네트워크 자원을 공유하면 측정값에 그 영향이 포함됩니다.

재현 가능한 결과를 비교하려면 실행 시각, 배포 사양, 참가자 수, 요청 간격, 실행 시간, 부하 발생기 위치와 서버 위치를 함께 기록해야 합니다.

현재 저장소에는 실제 수치가 담긴 Verified Run이 없으므로 예제 수치를 제시하지 않습니다.
