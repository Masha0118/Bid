## 실행하지 않은 이유

Gradle과 Flutter 테스트는 프로젝트 내부에 빌드·도구 산출물을 만들 수 있습니다. 이번 요청은 원본 프로젝트에 파일을 추가하거나 변경하지 말라는 조건이 우선이므로 실행하지 않았습니다. 실행 성공 건수, 성공률과 성능 수치를 기재하지 않습니다.

## 권장 재현 순서

원본 저장소 변경이 허용된 별도 복제 환경에서 다음 순서로 확인할 수 있습니다.

1. Auction Server 단위 테스트와 Kafka 통합 테스트
2. Main API 거래 중복 방지·보안 회귀 테스트
3. Flutter 실시간 상태 테스트
4. Batch 위험 분석과 PostgreSQL 통합 테스트
5. Redis, Kafka, RabbitMQ를 포함한 종료·복구 시나리오

## 공개 Flow Console 검증

Flow Console은 원본 서비스 테스트와 별개입니다. 이 공개 저장소에서 `npm run type-check`, `npm run test`, `npm run build`를 실행해 딥링크와 정적 빌드를 확인하며, 결과는 `테스트 실행 확인`으로 표시합니다. 이는 실제 서버, 네트워크와 장애 환경의 검증을 의미하지 않습니다.
