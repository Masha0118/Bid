# 보안

웹 체험은 제한된 경매 시나리오만 실행하고 내부 시스템으로 임의 요청을 전달하지 않도록 입력과 실행 경계를 좁힙니다.

## 입력과 실행 제한

- 가상 참가자 수를 허용 범위로 제한합니다.
- 입찰 속도와 실행 시간을 서버의 제한값 안에서만 선택합니다.
- 요청 금액의 형식과 범위를 검증합니다.
- 한 번에 실행할 수 있는 테스트 수를 제한합니다.
- 종료 후 다음 실행까지 안전 대기 시간을 적용합니다.
- 브라우저가 대상 URL, 내부 경로, 헤더, 명령이나 스크립트를 입력하지 못하게 합니다.

## 세션과 테스트 데이터

- 경매 체험 세션은 만료 시간을 가집니다.
- 테스트 사용자 인증 정보는 서버 내부에서만 사용합니다.
- 화면에는 참가자를 `나` 또는 가상 참가자 이름으로 표시합니다.
- 세션 종료 후 테스트 경매와 관련 데이터를 정리합니다.
- 정리 실패를 성공으로 숨기지 않고 실행 상태에 반영합니다.

## 요청 경계

- CORS는 확인된 웹 Origin만 허용하도록 구성합니다.
- Rate Limit으로 반복 요청을 제한합니다.
- 내부 API는 별도 인증값과 전용 실행 프로필로 보호합니다.
- 브라우저 응답과 실시간 이벤트에 내부 인증값을 포함하지 않습니다.
- 공개 화면에 내부 사용자 식별자와 시스템 경로를 직접 노출하지 않습니다.

## Secret 관리

인증값은 배포 환경의 Secret으로 주입하고 저장소 문서, 결과 파일과 화면에 기록하지 않습니다. 로그를 남길 때도 요청 헤더와 인증값을 마스킹하고, 테스트 결과에는 공개에 필요한 지표만 포함합니다.

## 격리와 영향 범위

체험용 경매 데이터와 메시징 경로는 제한된 테스트 환경에서 실행합니다. 실행 제어는 허용된 시나리오와 고정된 내부 대상만 사용하며, 브라우저가 다른 서버를 선택할 수 없습니다.

배포 검증에서는 CORS, Rate Limit, 동시 실행 잠금, 세션 만료와 데이터 정리가 실제 설정과 일치하는지 함께 확인해야 합니다.
