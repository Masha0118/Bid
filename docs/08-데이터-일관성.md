# 데이터 일관성

정합성 검사는 단순한 성공 표시가 아니라 서로 관련된 값이 같은 결과를 가리키는지 비교합니다.

| 확인 항목 | 비교 대상 | 의미 | 웹에서 확인 |
|---|---|---|---|
| 최종 최고가 | 반영된 가장 높은 입찰 금액 ↔ 최종 경매 가격 | 가격 판정 결과 일치 | 고급 결과에서 확인 |
| 최종 최고 입찰자 | 최고 금액의 참가자 ↔ 최종 최고 입찰자 | 가격과 참가자 연결 일치 | 고급 결과에서 확인 |
| 입찰 횟수 | 반영된 입찰 수 ↔ 경매 입찰 횟수 | 상태 변경 횟수 일치 | 부분 확인 |
| 상태 Version | 서버 최종 Version ↔ 참가자 카드 Version | 실시간 화면 최신 상태 일치 | 기본 체험에서 확인 |
| 완료 요청 수 | 시도한 입찰 ↔ 완료된 응답 | 처리 중 요청 존재 여부 | 기본 체험에서 확인 |
| Kafka 이력 | 전달 요청 ↔ 소비·저장 결과 | 비동기 이력 완료 여부 | 전달 요청까지만 부분 확인 |
| 최종 저장 데이터 | Redis 최종 상태 ↔ PostgreSQL 상태 | 런타임·영속 상태 수렴 | 기본 체험 미포함 |
| 거래 중복 | 동일 낙찰 결과 ↔ 생성된 거래 | 종료 재실행 시 중복 방지 | 자동화 테스트 확인 |

## 결과 해석

- 가장 높은 반영 입찰과 최종 경매 가격이 일치하는지 확인합니다.
- 그 금액을 제출한 참가자와 최종 최고 입찰자가 일치하는지 확인합니다.
- 각 참가자 카드의 Version이 서버의 최종 Version과 일치하는지 확인합니다.
- 시도한 입찰 수와 완료 응답 수가 다르면 아직 처리 중이거나 응답을 받지 못한 요청이 있는지 확인합니다.
- Kafka 전달 요청이 보인다는 사실만으로 최종 저장 완료를 판단하지 않습니다.

## PASS 외의 상태

- 확인 정보 없음: 비교에 필요한 값이 수집되지 않았습니다.
- 응답 시간 초과: 제한 시간 안에 비교 결과를 얻지 못했습니다.
- 추가 검토 필요: 값은 있지만 자동 판정만으로 원인을 확정하기 어렵습니다.
- 불일치: 기대 값과 실제 값이 다르며 실행 기록을 확인해야 합니다.

정합성 결과는 해당 실행과 테스트 환경의 상태를 설명합니다. 장기간 안정성이나 모든 장애 상황을 일반화하지 않습니다.
