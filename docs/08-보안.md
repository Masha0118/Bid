# 보안

코드에서 확인된 기능만 일반화해 정리했습니다. 실제 비밀값, 헤더·경로·허용 출처와 보안 임계값은 공개하지 않습니다.

## 인증과 세션

- Main API, Auction Server와 Admin Server에 Spring Security 필터 체인이 있습니다.
- JWT를 검증하고 서버 세션을 만들지 않는 인증 정책을 사용합니다.
- 관리자 인증 토큰과 사용자 인증 흐름은 분리돼 있습니다.
- 비밀번호 저장에는 BCrypt가 사용되며 비용 설정 범위를 검증합니다.

## 서버 간 요청

내부 서버용 요청은 일반 사용자 JWT와 별도의 공유 비밀 검증 필터를 거칩니다. 공개 문서에서는 실제 헤더 이름, 값의 구조와 내부 엔드포인트를 제외했습니다.

## 권한과 데이터 소유권

- 사용자 요청의 경로·본문 식별자가 인증 사용자와 일치하는지 확인하는 필터가 있습니다.
- 거래 결제·배송·상태 변경에서 구매자 또는 판매자 역할을 서비스 계층에서 다시 확인합니다.
- 관리자 요청은 별도 인증과 역할 상태를 확인합니다.
- 정지 계정과 이메일·전화 인증 상태에 따라 거래·입찰 관련 요청을 제한하는 정책 코드가 있습니다.

## 요청 보호

- Redis 기반 요청 횟수 제한과 Redis 장애 시 사용할 메모리 카운터 구현이 있습니다.
- 신뢰할 프록시 여부에 따라 전달된 클라이언트 주소를 사용할지 결정합니다.
- 서버별 CORS 허용 출처를 설정으로 제한합니다.
- 업로드 파일은 확장자만 보지 않고 실제 파일 시그니처와 콘텐츠 유형을 확인합니다.

## 오류 응답

인증 및 권한 오류를 일관된 JSON 응답 형식으로 반환하며, 인증 실패와 권한 부족을 서로 다른 HTTP 상태로 구분합니다. 요청 제한도 별도 상태로 응답합니다.

## 현재 검증 범위

보안 필터와 설정 코드, 비밀번호 정책, 요청 제한 카운터, 클라이언트 식별, 이미지 검증과 거래 상태 권한 테스트 파일을 확인했습니다. 이번 분석에서는 테스트를 실행하지 않았으며, 침투 테스트나 외부 보안 감사 결과를 주장하지 않습니다.
