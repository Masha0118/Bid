# 경매 도메인 규칙

아래 규칙은 현재 코드와 테스트 파일에서 확인한 범위만 정리했습니다. 악용 가능성이 있는 내부 임계값은 일반화했습니다.

| 규칙 | 확인된 처리 | 확인 수준 |
|---|---|---|
| 경매 시작 예약 | 상품 등록 시 시작·종료 예약 작업을 발행하고 경매 서버가 시작 상태를 구성 | 구현, 관련 테스트 코드 확인 |
| 시작가·입찰 단위 | 현재 최고가에 입찰 단위를 더해 최소 입찰가 계산 | 구현, 테스트 코드 확인 |
| 판매자 본인 입찰 | 판매자와 입찰자가 같으면 거절 | 구현, 테스트 코드 확인 |
| 최고 입찰자의 연속 입찰 | 현재 최고 입찰자의 추가 입찰을 거절 | 구현, 테스트 코드 확인 |
| 입찰 취소 | 현재 최고 입찰자만 최신 입찰을 취소할 수 있고 이전 최고 입찰을 복원 | 구현, 테스트 코드 확인 |
| 취소 후 재입찰 | 취소한 사용자의 같은 경매 재입찰을 차단 | 구현, 테스트 코드 확인 |
| 차단 관계 | 사용자 간 차단 관계가 있으면 입찰 자격 검증 단계에서 제한 | 구현, 관련 정책 코드 확인 |
| 마감 임박 자동 연장 | 설정이 켜져 있고 연장 조건과 최대 횟수 안이면 종료 시각과 횟수를 함께 변경 | 구현, Lua 원자 범위 확인 |
| 경매 종료 | 종료 메시지와 주기적 만료 재탐색 경로 제공 | 구현, 테스트 코드 확인 |
| 낙찰·유찰 | 입찰 기록으로 최종 입찰자를 결정하거나 유찰 상태 반영 | 구현, 테스트 코드 확인 |
| 거래 생성 | 낙찰 결과 반영 시 기존 거래를 먼저 확인해 생성 또는 재사용 | 구현, 테스트 코드 확인 |

## 상태별 책임

- 입찰 가능 여부와 현재 최고가는 Auction Server의 Redis 런타임 상태가 즉시 판정합니다.
- 상품의 낙찰·유찰 상태와 거래는 Main API의 PostgreSQL 영속 데이터에 반영합니다.
- 경매 종료 결과 요약과 입찰 감사 기록은 재처리와 확인을 위한 별도 영속 기록으로 남깁니다.

## 숫자 정책 공개 기준

입찰 단위, 종료 시각, 자동 연장 허용 여부는 각 경매의 데이터로 관리됩니다. 연장 시간과 최대 횟수의 내부 기본값은 코드에서 확인되지만 정책 악용과 변경 가능성을 고려해 공개 문서에는 고정 숫자로 적지 않습니다.
