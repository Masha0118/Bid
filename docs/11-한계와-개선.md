# 추가 검증 및 개선 과정

## 동시 입찰 부하 테스트

- 현재 확인: 관리자 개발 도구에 부하 실행과 리포트 코드, 경매 서버에 처리 지표가 있음
- 미확인: 재현 가능한 기준 환경의 결과와 장시간 안정성
- 검증 방법: 격리 환경에서 동일 경매·다중 경매 시나리오를 나누고 정합성 오류를 우선 측정
- 개선 방향: 평균 지연보다 실패 코드, 버전 연속성, DB 동기화 지연을 함께 관찰

## WebSocket 재연결·중복·역순·누락

- 현재 확인: 버전 비교 로직과 단위 테스트 시나리오
- 미확인: 실제 네트워크 단절과 다중 서버 전환
- 검증 방법: 연결 강제 종료, 이벤트 지연·중복·삭제를 주입하는 통합 테스트
- 개선 방향: 재동기화 횟수와 원인을 지표로 남기고 과도한 전체 조회 방지

## Redis 재시작과 상태 복원

- 현재 확인: 영속 백업에서 누락 상태를 복원하고 만료 경매를 재탐색하는 코드
- 미확인: 재시작 중 입찰과 복원 순서, 부분 데이터 손상
- 검증 방법: 경매 진행 중 Redis 재시작 후 버전·최고가·입찰 기록 비교
- 개선 방향: 복원 검증 체크섬 또는 상태 비교, 복구 중 입찰 차단 정책 명확화

## 실시간 상태와 영속 데이터 동기화

- 현재 확인: 현재가·입찰 기록을 비동기 후속 경로로 PostgreSQL에 반영
- 미확인: 후속 큐가 장시간 실패할 때의 손실 범위
- 검증 방법: Main API·DB 장애 주입 뒤 재시도와 최종 데이터 비교
- 개선 방향: 공유 가능한 영속 작업 로그 또는 Outbox 계열 패턴 검토

## 전체 서버 CI 테스트 안정화

- 현재 확인: 보안·의존성 검사 Workflow와 서버별 테스트 코드
- 미확인: 현재 HEAD의 전체 CI 성공 결과
- 검증 방법: 깨끗한 복제 환경에서 서버·Flutter 테스트와 Testcontainers 실행
- 개선 방향: 단위·통합·인프라 테스트를 분리하고 실패 원인을 산출물로 보존

## 로그와 지표 시각화

- 현재 확인: 처리 지연, 비동기 큐와 비즈니스 지표, Prometheus 연동 코드
- 미확인: 경보 기준과 장기간 추세
- 검증 방법: 입찰부터 거래 생성까지 상관관계 식별자로 추적
- 개선 방향: 종료 재시도, 버전 누락, 로컬 실패 보관 크기를 핵심 지표로 정의

## 이미지 저장 구조

- 현재 확인: 로컬 파일 저장, 임시 업로드·초안 정리와 보안 검증
- 미확인: 다중 서버 공유와 대용량 미디어 수명주기
- 검증 방법: 중복 업로드, 정리 실패, 서버 교체 시 파일 접근 시험
- 개선 방향: 객체 저장소와 메타데이터 책임 분리, 삭제 정책 자동화

## AI 기능 품질 평가

- 현재 확인: 상품 등록 초안과 고객센터 검색·답변 코드 및 일부 단위 테스트
- 미확인: 실제 사용자 질문의 정확도·안전성·비용
- 검증 방법: 비식별 평가 세트로 정확성, 근거성, 실패 응답을 반복 측정
- 개선 방향: 기능 노출 조건과 사람 검토 경로를 유지하고 품질 기준 충족 후 범위 확대
