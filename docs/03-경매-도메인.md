# 경매 도메인

## 경매 상태

경매는 시작 전, 진행 중, 종료 처리 중, 종료 완료 또는 취소 상태로 구분됩니다. 입찰은 진행 중이고 종료 시각이 지나지 않은 경매에서만 판정합니다.

## 가격과 입찰 조건

- 시작 가격: 경매가 처음 시작되는 기준 금액입니다.
- 입찰 단위: 현재가에서 최소로 올려야 하는 금액입니다.
- 최고 입찰자: 현재까지 반영된 가장 높은 입찰의 참가자입니다.
- 연속 입찰 제한: 이미 최고 입찰자인 참가자의 요청은 조건상 미반영될 수 있습니다.
- 낮은 금액 제한: 최소 다음 입찰가보다 낮은 요청은 반영하지 않습니다.
- 종료 제한: 종료된 경매에는 새 입찰을 반영하지 않습니다.

웹 체험은 현재가와 입찰 단위를 기준으로 최소 다음 입찰가를 표시합니다. `+1천`, `+5천`은 입력 편의를 위한 기능이며 최종 판정은 서버가 수행합니다.

## 입찰이 반영될 때 바뀌는 상태

- 현재가
- 최고 입찰자
- 입찰 횟수
- 상태 Version
- 마감 임박 조건을 만족한 경우 종료 시각과 연장 횟수

입찰 가능 여부 확인과 위 상태의 Redis 변경은 Lua의 한 실행 안에서 수행됩니다. 메시지 전달과 영속 저장은 그 이후의 후속 처리입니다.

## 자동 연장

마감 직전의 유효한 입찰은 정책에 따라 종료 시각을 연장할 수 있습니다. 연장 여부와 횟수는 입찰 판정과 함께 갱신됩니다. 기본 웹 체험은 짧은 세션의 직접·가상 입찰에 집중하므로 전체 예약 종료 흐름과 자동 연장 정책을 모두 노출하지는 않습니다.

## Version

Version은 경매 상태가 유효하게 바뀔 때 증가합니다. 참가자 화면은 Version을 비교해 중복·역순·누락 가능성을 판단합니다. 값이 크다는 사실만으로 성공을 의미하지 않으며, 같은 경매의 상태 순서를 판정하는 기준으로 사용합니다.

## 경매 종료

예약된 종료 작업이 도착하면 경매별 잠금을 얻고 이미 완료된 결과가 있는지 확인합니다. Redis 런타임 상태를 우선 사용하고 필요한 경우 영속 데이터를 보완 조회합니다. 후속 결과 반영이 끝난 뒤 런타임 상태를 정리합니다.
